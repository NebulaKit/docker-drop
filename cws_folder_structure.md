# DROP Cloud Workstation Directory Layout

This is the recommended directory structure when preparing input files and running DROP v1.4.0 on DNAnexus Cloud Workstation.

```text
drop_analysis/
│
├── data/
│   ├── vcf_files/
│   │   ├── file_name.vcf.gz
│   │   ├── file_name.vcf.gz.tbi
│   │   └── ...
│   ├── bam_files/
│   │   ├── file_name.bam
│   │   ├── file_name.bam.bai
│   │   └── ...
│   ├── GRCh38.no_alt_analysis_set.fa
│   ├── GRCh38.no_alt_analysis_set.fa.fai
│   ├── gencode.v40.annotation.gtf
│   ├── qc_vcf_1000G_hg38.vcf.gz
│   ├── qc_vcf_1000G_hg38.vcf.gz.tbi
│   ├── chosen-config.yaml
│   └── chosen_sample_annotation.tsv
│
├── analysis/         # Intermediate files generated by DROP
├── output/           # Final output results from DROP
```

> 📁 Tip: Mount `drop_analysis/` as `/drop` inside your Docker container, and ensure that the `config.yaml` correctly points to the relevant file paths under `/drop/data/`.
